Topic 1: Memory Hierarchy and Locality of Reference

Memory hierarchy is a layered arrangement of storage from fastest/smallest/most expensive to slowest/largest/cheapest.

Typical hierarchy order (top to bottom)

Registers -> L1 Cache -> L2 Cache -> L3 Cache -> Main Memory (DRAM) -> Secondary Storage (SSD/HDD) -> Optical/Tape

Locality of reference

Locality of reference means programs tend to access a small set of memory locations repeatedly over short intervals.

Temporal locality

Recently used data/instruction is likely to be used again soon.

Example: loop counter variable used every iteration.

Spatial locality

Locations near recently accessed location are likely to be used soon.

Example: reading array elements in sequence.

Exam Focus:
  • As we go down hierarchy: capacity increases, cost/bit decreases, access time increases, access frequency decreases.
  • Always connect locality to why cache works.

Topic 4: Main Memory Technologies (SRAM, DRAM, SDRAM, RDRAM, DDR)

SRAM (Static RAM)

  • Stores bit using flip-flop latch.
  • No periodic refresh needed.
  • Very fast, low density, expensive.
  • Used mainly in cache memory.

DRAM (Dynamic RAM)

  • Stores bit as charge in capacitor.
  • Requires periodic refresh.
  • Slower than SRAM but high density and cheaper.
  • Used mainly in main memory.

Modern DRAM families

Type Key idea Use/impact
SDRAM Synchronous with system clock Predictable timing, pipelined operations
RDRAM High-speed channel architecture (Rambus) Higher bandwidth in older systems, cost/heat concerns
DDR SDRAM Transfers data on both rising and falling clock edges Higher throughput; DDR2/3/4/5 generations

Topic 6: Virtual Memory Basics and Relation with Cache

Virtual memory gives each process a large logical address space by mapping virtual pages to physical frames.

Core components

  • Virtual address split into page number and offset.
  • Page table maps virtual page to physical frame.
  • TLB stores recent page table entries for fast translation.
  • If page missing from RAM, page fault occurs and OS brings page from disk.

Virtual memory translation diagram

Virtual memory vs cache (similarity and difference)

Point Virtual memory Cache
Main goal Increase logical memory space and isolation Reduce average access latency
Transfer unit Page Block/line
Hardware support TLB + page table + OS Tag store + mapping/replacement logic
Miss handling Page fault (OS involvement) Cache fill (hardware managed)
Underlying principle Both exploit locality of reference

Topic 7: Error Detection and Correction (Hamming Code)

Hamming code adds parity bits at power-of-two positions to detect and correct single-bit errors.

How many parity bits are needed?

Find smallest r such that: 2^r >= m + r + 1 where m = number of data bits

Standard procedure (exam writing steps)

  1. Compute r and total code length n = m + r.
  2. Place parity bits at positions 1, 2, 4, 8, ... (position counted from rightmost bit as 1).
  3. Fill data bits in remaining positions.
  4. Compute each parity bit (even or odd parity according to question).
  5. At receiver, compute syndrome bits S1, S2, S4, S8, ...
  6. Syndrome value gives error position; flip that bit to correct.

Coverage sets for parity bits

P1 checks positions with LSB=1: 1,3,5,7,9,11,13,... P2 checks positions with second bit=1: 2,3,6,7,10,11,14,... P4 checks positions with third bit=1: 4,5,6,7,12,13,14,... P8 checks positions with fourth bit=1: 8,9,10,11,12,13,14,...

PYQ 2019 Q4(a): What are the key characteristics of computer memory systems?

Exam-ready answer:

The key characteristics are location, capacity, unit of transfer, access method, performance, physical type, physical characteristics, and organization.

Location indicates where memory is placed (register, cache, main, external). Capacity tells storage amount (word size and number of words/bytes). Unit of transfer defines how much data moves per operation. Access method can be sequential/direct/random/associative. Performance includes access time, cycle time, and transfer rate. Physical type denotes semiconductor/magnetic/optical classes. Physical characteristics include volatility and erasability. Organization describes arrangement of bits and words.

PYQ 2019 Q4(b) and 2023 Q4(b): Correct trends in memory hierarchy.

Given incorrect trend set in question: decreasing capacity, decreasing cost/bit, increasing frequency, increasing access time.

Correct statement as we move downward in hierarchy: 1) Capacity increases (not decreases). 2) Cost per bit decreases. 3) Frequency of CPU access decreases. 4) Access time increases.

Fill-in answer (2023 style): decreasing cost/bit, increasing capacity, and increasing access time.

PYQ 2022 Q4(a): Why not use one single fastest memory instead of hierarchy?

Model answer:

A single fastest memory (like all-SRAM system) would be prohibitively expensive, high in power consumption, and physically large for practical capacities. Hierarchy gives balanced design: small fast memory near CPU for frequent accesses and large cheap memory for bulk storage. Locality makes this efficient.

Therefore hierarchy provides near-fast performance at affordable cost and practical capacity.

PYQ 2019 and 2022: Features of DRAM and SRAM; key functions of SDRAM, RDRAM, DDR.

Model answer structure:

  • DRAM: capacitor-based, requires refresh, high density, low cost, used as main memory.
  • SRAM: flip-flop based, no refresh, very fast, lower density, high cost, used in cache.
  • SDRAM: synchronous with system clock; supports pipelined burst access.
  • RDRAM: high-speed channel interface; designed for high bandwidth in older platforms.
  • DDR: double data rate transfer on both clock edges, increasing effective throughput.

Extra high-mark point: DDR generations improve bandwidth and reduce per-bit energy over time.

PYQ 2021 Q2(c) and 2020(c): Key factors for virtual memory and similarity with cache.

Model answer:

  • Important factors: page size, replacement policy, write policy, TLB size/associativity, page table structure, disk I/O latency, page fault rate.
  • Virtual memory and cache both exploit locality and both keep a faster subset near CPU of a larger slower store.
  • Difference: virtual memory transfers pages and may involve OS page fault; cache transfers blocks and usually hardware handles misses.

PYQ 2019(c) and 2023(d): Define locality of reference.

Model answer:

Locality of reference is the tendency of programs to access a small set of instructions/data repeatedly (temporal locality) and nearby addresses in clusters (spatial locality). This property enables cache and virtual memory to improve average performance significantly.

PYQ Hamming 2019: data=1001010101, bit-10 changed. Find code and correct error.

Assumption: Even parity, rightmost bit position = 1.

m=10. Find r such that 2^r >= m+r+1. r=4 since 16 >= 15. Total length n=14.
Computed codeword = 10010110100101.
If bit position 10 is flipped, received word becomes 10011110100101. Syndrome calculation gives S=10. Therefore error is at bit 10 from right; flipping it restores 10010110100101.
PYQ Hamming 2022: data=1001110101, bit-8 changed. Find code and correct error.

Assumption: Even parity, rightmost bit position = 1.

m=10, r=4, n=14. Codeword = 10011100100111.
If bit-8 flips, received = 10011110100111. Syndrome S=8, so error position is 8 (from right). Corrected codeword = 10011100100111.
PYQ Hamming 2023: data=11001111 (odd parity), bit-7 changed. Find code and correct error.

Given: Odd parity and rightmost position counting.

m=8, r=4, n=12. Codeword (odd parity) = 110011111111.
After bit-7 flip, received word = 110010111111. Syndrome value S=7. So bit-7 is erroneous; flip it back. Correct codeword = 110011111111.
PYQ 2023 Q4(c) style numerical: 90% cache hit, T1=0.03 us, T2=0.2 us. Average access time.
Ta = T1 + (1-H)T2 H = 0.9 Ta = 0.03 + (0.1)(0.2) Ta = 0.03 + 0.02 = 0.05 us

Final answer: Average access time = 0.05 microseconds.

Topic 1: Need for Cache - Processor-Memory Speed Gap

Cache memory is a small, fast memory between CPU and main memory that stores recently/frequently used blocks.

Cache is the key architectural response to the memory wall problem. CPU cores can execute operations much faster than main memory can deliver operands, so without cache, execution units remain idle for many cycles.

By storing likely-to-be-reused blocks close to CPU, cache converts many slow memory accesses into fast accesses. This reduces average latency and improves effective processor utilization.

Why cache is needed

  • CPU speed has increased much faster than DRAM latency.
  • Without cache, CPU stalls frequently waiting for memory.
  • Locality of reference allows a small fast memory to serve most accesses.

Exam-quality explanation should connect all three points: speed gap creates wait, locality enables prediction of reuse, and cache uses that reuse to hide most memory delay.

Topic 2: Cache Operation (Read/Write Flow) and Key Terminology

Cache operation is a lookup-and-validate process. Address bits are split so hardware can quickly select candidate line/set, compare tag identity, and decide hit or miss.

Correct understanding of this flow is essential for mapping and performance numericals, because hit ratio and miss penalty are direct outcomes of these lookup decisions.

Core terms

Term Meaning
Cache hit Requested block found in cache
Cache miss Requested block not found in cache
Hit ratio (H) Fraction of accesses that hit in cache
Miss ratio 1 - H
Miss penalty Extra delay to fetch block from lower level
Cache line / block Unit transferred between cache and lower memory
Tag Identifier to determine which memory block is in a line

Read operation flow

1) CPU sends address 2) Index selects candidate line/set 3) Tag comparison 4a) If hit: return data quickly 4b) If miss: fetch block from lower memory, update cache, return data

Write operation summary

Write-through

Write both cache and main memory on every write. Simpler coherence, more traffic.

Write-back

Write cache first; update memory on eviction if dirty. Lower traffic, coherence complexity.

Write policy selection is a trade-off: write-through simplifies consistency but increases memory traffic, while write-back improves bandwidth efficiency but requires dirty-bit management and stronger coherence control.

  • For flowchart question, draw hit and miss branches clearly.
  • Define miss penalty separately from miss ratio.

Topic 3: Mapping Methods (Direct, Fully Associative, Set Associative)

Mapping function defines where a main memory block can be placed in cache.

Mapping method controls conflict behavior and hardware complexity. Direct mapping minimizes hardware cost but can force unrelated blocks to compete for one line. Fully associative minimizes conflict but requires expensive parallel tag comparison. Set-associative design is the practical middle ground.

In real processors, mapping choice is tied to target workload, latency goals, and silicon budget. That is why many designs use direct or low-way set-associative structures in L1 and higher associativity in lower levels.

Mapping Placement rule Pros Cons
Direct Each block maps to exactly one line (i = j mod m) Simple, fast, low cost High conflict misses
Fully associative Block can go to any line Lowest conflict miss Expensive associative search
k-way set associative Block maps to one set; any of k lines inside set Balance of cost and miss rate More complex than direct

Topic 4: Address Split Problems (Tag, Line/Set, Word/Offset Bits)

Address-split problems are bit-allocation problems. Once memory, cache, and block sizes are expressed as powers of two, field widths become straightforward logarithmic counts.

Most errors happen when unit conversion is skipped. Always convert MB/KB to bytes first, then compute total-address bits, line/set bits, and finally tag bits.

General formulas

Given: Main memory size = 2^N bytes Cache size = 2^C bytes Block size = 2^B bytes Direct mapping: Offset bits = B Number of lines = 2^(C-B) Line bits = C-B Main memory blocks = 2^(N-B) Tag bits = (N-B) - (C-B) = N-C k-way set associative: Number of sets = cache_lines / k Set bits = log2(number_of_sets) Tag bits = (N-B) - Set bits

Topic 5: Performance Terms and Core Formulas

Cache performance is probabilistic. Average access time combines fast hit path and slower miss path weighted by their probabilities.

This is why improving hit ratio by even a few percent can significantly reduce average latency when miss penalty is large.

Term Meaning Formula
Hit ratio Probability of hit in cache H = hits / total accesses
Miss ratio Probability of miss 1 - H
Average access time Expected memory access delay Ta = Tc + (1-H)Tm
Miss penalty Extra miss delay Tm (or additional lower-level time)

Derivation of Ta = Tc + (1-H)Tm

Expected time Ta = H(Tc) + (1-H)(Tc + Tm) = HTc + (1-H)Tc + (1-H)Tm = Tc + (1-H)Tm

Interpretation: Tc is paid for every access, while Tm is paid only on misses. Therefore miss ratio directly scales the expensive part of delay.

Two-level cache relation

For L1 miss going to L2: Ts = T1 + (1-H1)T2 Access efficiency form: T1/Ts = 1 / [1 + (1-H1)(T2/T1)] If H1 is high, Ts approx T1.

Topic 6: Numerical Problem Workshop

Numerical questions in this module generally fall into repeatable templates. If you identify the template first (bit split, hit/miss timing, or stall impact), solving becomes systematic and fast.

Fast solving checklist

  1. Normalize all sizes to powers of two in bytes.
  2. Compute intermediate counts (lines/blocks/sets) before tag bits.
  3. Write formula symbolically once before substitution.
  4. State assumptions clearly when workload details are missing.

Problem type A: Tag bits and directory size

Example (PYQ 2022): Cache = 128KB = 2^17 bytes Block = 4KB = 2^12 bytes Main memory = 128MB = 2^27 bytes Lines = 2^(17-12) = 2^5 = 32 Main memory blocks = 2^(27-12) = 2^15 Tag bits = 15 - 5 = 10 Directory size (tags only) = 32 x 10 = 320 bits (If valid bit included: 32 x 11 = 352 bits)
Example (PYQ 2023): Cache = 256KB = 2^18 Block = 8KB = 2^13 Main memory = 512MB = 2^29 Lines = 2^(18-13)=2^5 Main memory blocks = 2^(29-13)=2^16 Tag bits = 16 - 5 = 11

Problem type B: Hit-ratio based average time

If 95% accesses hit: Ta = Tc + (1-0.95)Tm = Tc + 0.05Tm

Problem type C: Cache miss penalty impact on CPI

Stall CPI contribution = miss_rate x miss_penalty x accesses_per_instruction Total CPI = base_CPI + stall_CPI Speedup(perfect cache) = CPI_with_miss / CPI_perfect
In miss-penalty numericals, explicitly state any missing assumption such as number of data memory accesses per instruction.

Topic 7: Replacement Policies and Write Policies

Replacement and write policies determine cache behavior under pressure. Mapping tells where blocks can go, policy tells which block leaves and how memory consistency is maintained on writes.

Replacement policies

Policy Idea Strength Limitation
LRU Replace least recently used line Good practical hit behavior Tracking overhead
FIFO Replace oldest inserted line Simple May evict useful line
LFU Replace least frequently used line Frequency aware Extra counters needed
Random Random line replacement Very simple hardware Unpredictable quality

Write policies

Write-through

  • Write to cache and main memory together.
  • Simple consistency with memory.
  • High write traffic and slower writes.

Write-back

  • Write to cache first; mark dirty.
  • Main memory updated on eviction.
  • Lower traffic but coherence complexity.

Practical systems may combine write policies with allocation strategies (write-allocate or no-write-allocate), so exam answers become stronger when this interaction is mentioned briefly.

Exam Focus:
  • Direct mapping has no replacement choice.
  • Mention dirty bit while discussing write-back.
  • For multi-core context, mention cache coherency implications.

Topic 8: Multi-level Cache, Unified vs Split, Coherency Basics

Multi-level cache architecture balances speed and capacity across levels: very small and very fast L1 near core, larger but slower lower levels. Effective design focuses on minimizing average access time rather than optimizing only one level.

Why multi-level cache

  • L1 very fast but small.
  • L2/L3 larger but slower, still much faster than DRAM.
  • Hierarchy balances latency, capacity, and cost.

Unified vs split cache

Type Advantage Limitation
Unified Single pool balances instruction/data demand, simpler design Contention between instruction and data accesses
Split I-cache + D-cache Parallel instruction fetch and data access, better pipeline support More design complexity

Cache coherency basics

Cache coherency ensures all processor caches see consistent values for shared memory locations.

In multicore systems, coherence is mandatory for correctness: a write by one core must become visible to other cores according to the memory consistency model. Coherence protocols enforce this through invalidation/update and state tracking.

  • Common approach: invalidate or update other cache copies on write.
  • Write-invalidate protocols are common due to lower traffic.
  • Directory/snooping approaches track shared block ownership state.
Exam Focus:
  • For coherency short note, define problem then mention invalidate/update approach.
  • For multilevel cache questions, mention L1/L2/L3 trade-off explicitly.

PYQ 2022 Q4(b): Significance of cache memory and cache read operation.

Exam-ready answer:

Cache memory bridges the CPU-main memory speed gap by storing frequently used blocks close to CPU. It reduces average memory access time and increases CPU utilization.

Read operation steps: 1) CPU sends address. 2) Cache index/set selected. 3) Tag compared. 4) On hit -> data returned quickly. 5) On miss -> block fetched from lower memory, cache updated, data returned.

This mechanism exploits locality of reference, hence a small cache can significantly improve overall performance.

PYQ 2019 Q4(c): Show that Ta = Tc + (1-H)Tm.
Ta = H(Tc) + (1-H)(Tc + Tm) = HTc + (1-H)Tc + (1-H)Tm = Tc + (1-H)Tm

where Tc is cache access time, Tm is miss penalty from lower memory, and H is hit ratio.

PYQ 2019 Section B(a): If Ta = Tc in Ta = Tc + (1-H)Tm, what is H? If Tc << Tm, what happens?
Ta = Tc + (1-H)Tm and Ta = Tc => (1-H)Tm = 0 => H = 1 (assuming Tm > 0)

If Tc << Tm, misses become very expensive; average time is highly sensitive to (1-H). Therefore very high hit ratio is required to keep Ta low.

PYQ 2019 Section B(d): If 95% accesses are found in cache, express average access time.
H = 0.95 Ta = Tc + (1-H)Tm Ta = Tc + 0.05Tm

That is the required expression.

PYQ 2019 Section B(b): How to measure performance of two-level cache memory? Deduce expression.

Model derivation:

Let T1 = L1 access time, H1 = L1 hit ratio, T2 = L2 access penalty (including lower level reference) Then average service time: Ts = T1 + (1-H1)T2 Efficiency form: T1/Ts = 1 / [1 + (1-H1)(T2/T1)]

This expression shows Ts approaches T1 when H1 is high.

PYQ 2022 Q5(b): Cache=128KB, block=4KB, main memory=128MB. Find tag bits and tag directory size.
Cache size = 2^17 bytes Block size = 2^12 bytes Main memory size = 2^27 bytes Lines = 2^(17-12) = 2^5 = 32 Main memory blocks = 2^(27-12) = 2^15 Tag bits = 15 - 5 = 10
Tag directory size (tag only) = 32 x 10 = 320 bits. If valid bit included, directory = 32 x (10+1) = 352 bits.
PYQ 2023 Q5(b): Cache=256KB, block=8KB, main memory=512MB. Find tag field bits.
Cache = 2^18, Block = 2^13, Main memory = 2^29 Lines = 2^(18-13) = 2^5 Main memory blocks = 2^(29-13) = 2^16 Tag bits = 16 - 5 = 11

Final answer: Tag field = 11 bits (direct mapping assumption).

Mapping Placement Conflict behavior Hardware cost
Direct One fixed line Highest conflict miss Lowest
Set associative One set, k choices Moderate conflict miss Moderate
Fully associative Any line Lowest conflict miss Highest
PYQ 2023 Q6(a): Show Ts approx T1 from efficiency relation.
Given: T1/Ts = 1 / [1 + (1-H)(T2/T1)] Rearrange: Ts = T1[1 + (1-H)(T2/T1)] Ts = T1 + (1-H)T2

If H is close to 1, (1-H) is very small, so second term is small. Therefore Ts approx T1.

PYQ 2020: What is cache miss penalty? Write handling steps.

Model answer:

Cache miss penalty is the extra delay incurred when requested data is absent in cache and must be fetched from lower-level memory.

Miss handling: 1) Detect miss after tag comparison. 2) Select victim line (if needed). 3) If victim dirty (write-back), write it to lower memory. 4) Fetch required block from lower level. 5) Fill cache line and update tag/valid bits. 6) Restart/complete original access.
PYQ 2020(c): Perfect cache speedup from miss rates and penalty.

Given: instruction miss rate 2%, data miss rate 4%, miss penalty 100 cycles, base CPI=2.

Assumption: one instruction fetch and one data access per instruction (if not specified).

Stall CPI = (0.02 x 100) + (0.04 x 100) = 2 + 4 = 6 CPI_with_miss = 2 + 6 = 8 CPI_perfect = 2 Speedup = 8/2 = 4

Final answer (under stated assumption): perfect cache gives 4x speedup.

PYQ 2022 Q6(c): Briefly explain approaches to cache coherency.

Model answer:

  • Write-invalidate: writer invalidates other cached copies, forcing later readers to refetch.
  • Write-update: writer updates other copies immediately.
  • Snooping protocols: caches monitor shared bus transactions to maintain consistency.
  • Directory-based protocols: directory tracks sharers and coordinates coherence in scalable systems.

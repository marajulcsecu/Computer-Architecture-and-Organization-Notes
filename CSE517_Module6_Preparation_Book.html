Topic 1: Pipeline Concept and Speedup

Pipelining divides instruction execution into stages so multiple instructions are processed concurrently, one per stage, like an assembly line.

Classic 5-stage instruction pipeline

IF
Instruction Fetch
ID
Decode/Register Read
EX
Execute/ALU
MEM
Memory Access
WB
Write Back

Core formulas

Let n = number of instructions k = number of pipeline stages t = stage time Non-pipelined time: Tnp = n x k x t Pipelined time: Tp = (k + n - 1) x t Speedup: S = Tnp / Tp = (n x k) / (k + n - 1) As n gets very large, S approaches k.

Important interpretation

  • Pipelining improves throughput more than single-instruction latency.
  • Speedup is reduced by hazards, stalls, and stage imbalance.
  • Ideal speedup is rarely achieved in real processors.
Exam Focus:
  • Always write fill and drain term (k + n - 1) in Tp.
  • Mention "throughput increases" in theoretical answers.

Topic 3: Pipeline Hazards and Stalls

Main hazard categories

Hazard Type Cause Typical Example Avoidance / Remedy
Structural Resource conflict in same cycle Single memory used by IF and MEM together Separate I-cache and D-cache, duplicate resources
Data Instruction depends on previous result lw r1,... followed by add r2,r1,r3 Forwarding, compiler scheduling, stalling
Control Branch/jump changes next PC Wrong-path fetched after branch Branch prediction, delayed branch, flush

Hazard detection example

I1: lw t1, 0(t0) I2: lw t2, 4(t0) I3: add t3, t1, t2 I4: sw t3, 12(t0) I5: lw t4, 8(t0) I6: add t5, t1, t4 I7: sw t5, 16(t0) Potential hazards: - load-use between I2 and I3 - RAW from I3 to I4 - load-use between I5 and I6

Stall-avoiding reorder (one practical schedule)

lw t1, 0(t0) lw t2, 4(t0) lw t4, 8(t0) add t3, t1, t2 add t5, t1, t4 sw t3, 12(t0) sw t5, 16(t0)
Exam Focus:
  • First identify hazard type, then write remedy; this gives full-credit structure.
  • For 2-mark questions, one example each for data/control hazards is enough.

Topic 5: Flynn Classification and Parallel Processors

Flynn taxonomy classifies computers by number of concurrent instruction streams and data streams.

Summary table

Class Instruction Streams Data Streams Example
SISD 1 1 Single-core scalar execution
SIMD 1 Many GPU, vector processors
MISD Many 1 Rare fault-tolerant niche systems
MIMD Many Many Multicore CPUs, distributed systems

Parallel processor notes (SIMD and MIMD)

  • SIMD is efficient for regular data-parallel workloads (matrix, image, ML vector kernels).
  • MIMD is flexible for task parallelism and heterogeneous workloads.
  • Scalability depends on interconnect, memory bandwidth, and synchronization overhead.
Exam Focus:
  • If asked theoretical-interest class: answer MISD and explain limited practical workload match.
  • For 2-mark questions, define SIMD and MIMD with one real example each.

Topic 6: Traditional vs High-Speed Bus Architecture

Traditional bus model

  • Single shared bus for CPU, memory, and I/O.
  • Simple design but high contention and limited bandwidth.
  • As CPU speed increases, shared bus becomes bottleneck.

High-speed bus model

  • Hierarchical buses or switched interconnect.
  • Dedicated high-speed processor-memory path.
  • Split transaction, pipelined transaction, and burst transfer support.
  • Modern examples use point-to-point links (for example PCIe style).

Diagrammatic difference

  • Use keywords: contention, throughput, scalability, dedicated path.

Topic 7: RAID Basics (0/1/2/3 + Brief Overview of Others)

RAID (Redundant Array of Independent Disks) combines multiple disks for performance, reliability, or both.
RAID Level Main Idea Fault Tolerance Performance Character
RAID 0 Block striping, no parity/mirror None Very high read/write throughput
RAID 1 Mirroring High (one copy survives) Fast reads, write duplicated
RAID 2 Bit-level striping with Hamming ECC disks ECC-based recovery Historically important, rarely used now
RAID 3 Byte-level striping + dedicated parity disk Survives one disk failure Good sequential throughput, parity bottleneck

PYQ 2020/2022: "The pipelined approach takes much less time" show with example.

Model answer: Consider n = 20 instructions, k = 5 stages, and stage time t = 1 ns.

Tnp = n x k x t = 20 x 5 x 1 = 100 ns Tp = (k + n - 1) x t = (5 + 20 - 1) x 1 = 24 ns Speedup S = Tnp/Tp = 100/24 = 4.167

So pipelined design completes the same instruction stream much faster. This validates the statement. For larger n, speedup gets closer to k.

PYQ 2020 A(k), 2021 A2(d), 2022 Q8(d): What are pipeline hazards and stalls? How to avoid?
Hazard Definition Avoidance
Structural Two stages need same hardware simultaneously Resource duplication, split caches
Data Instruction depends on unavailable prior result Forwarding, scheduling, stalling
Control Branch target/decision not known early Branch prediction, delayed branch, flush
A stall is a bubble inserted into the pipeline to preserve correctness when hazard cannot be resolved immediately.
PYQ 2020 B(c): Find hazards in given MIPS sequence and reorder to avoid stalls.

Given code (shortened): lw, lw, add, sw, lw, add, sw.

Hazards identified: 1) load-use hazard before first add. 2) load-use hazard before second add. 3) RAW dependency from add to store.
A stall-reduced schedule: lw t1, 0(t0) lw t2, 4(t0) lw t4, 8(t0) add t3, t1, t2 add t5, t1, t4 sw t3, 12(t0) sw t5, 16(t0)

This schedule separates dependent operations and reduces bubble insertions.

PYQ 2019 Q8(b): What is pipelining? What do you mean by SIMD and MIMD?

Exam-ready answer:

  • Pipelining is overlapped instruction execution using multiple stages.
  • SIMD executes one instruction over many data items in parallel.
  • MIMD executes different instructions on different data streams in parallel processors.

Example: SIMD for image pixel operations; MIMD for multicore task-parallel workloads.

PYQ 2023 Q8(a): Among SISD, SIMD, MISD and MIMD, which is mostly theoretical interest and why?

Answer: MISD is mostly of theoretical interest.

Reason: 1) Real workloads rarely apply multiple different instruction streams to exactly same data stream. 2) Hardware complexity is high with limited practical gain. 3) Only niche fault-tolerant or signal-processing style systems resemble it.

PYQ 2023 Q3(b) and 2019 Q3(c): Diagrammatically show traditional vs high-speed bus architecture.

Answer points to include in diagram:

  • Traditional: one shared bus among CPU, memory, I/O (high contention).
  • High-speed: hierarchical/segmented buses, dedicated memory path, better arbitration.
  • Result: higher bandwidth and improved scalability.

RAID Feature summary
0 Striping only, highest throughput, no fault tolerance.

3 Byte-level striping plus dedicated parity disk, good sequential bandwidth.
PYQ 2020 A(a): Define datapath and explain single-bus organization of a processor.

Datapath: The collection of functional units and interconnections that carry and transform data inside CPU (registers, ALU, buses, shifter, memory interface).

Single-bus organization: 1) A common internal bus interconnects CPU registers and functional units. 2) Control unit selects source register to place data on bus. 3) Destination register latches data from bus. 4) ALU operation and memory register transfers are sequenced over clock cycles.

Advantage: simpler wiring. Limitation: one transfer at a time creates bandwidth bottleneck.

PYQ 2020 A(h)/2021 B(b): Construct a 1-bit ALU using basic logic gates.

Model answer: Build logic block (AND, OR), full-adder block, and select output through MUX using control lines.

Inputs: A, B, Cin, controls S1,S0 Logic outputs: L0 = A AND B, L1 = A OR B Arithmetic output: Sum = A + B + Cin Subtraction using two's complement: A + (NOT B) + 1 Output F selected by control through 4-to-1 MUX

Cascading this cell forms 4-bit, 8-bit, 32-bit ALU designs.

PYQ 2021 A(d): Write five stages of MIPS pipeline.

Answer: IF, ID, EX, MEM, WB.

IF: fetch instruction ID: decode and read registers EX: ALU execute / address computation MEM: data memory access WB: write result back to register file

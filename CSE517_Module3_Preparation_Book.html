Topic 1: Basic Instruction Cycle and Register Transfer Flow

Instruction cycle: The complete process by which CPU fetches one instruction, decodes it, executes it, and updates state for the next instruction.

Instruction cycle is the bridge between program code and hardware action. A program looks like a static sequence of instructions in memory, but CPU turns that sequence into real behavior only by repeatedly running this cycle.

Correctness depends on strict ordering of micro-operations: instruction address must be formed before memory read, instruction must be decoded before control signals are issued, and destination state must be updated only after execution result is ready.

Performance analysis in later modules also depends on this model, because pipelining, hazards, and CPI are all extensions of how this basic cycle is organized and overlapped.

Main stages

Fetch

Instruction is read from memory using PC.

Decode

Control unit identifies operation and addressing mode.

Execute / Write back

ALU, memory, or control action runs and result is stored.

Why each stage exists

  • Fetch isolates memory access and keeps program order under PC control.
  • Decode translates opcode bits into concrete control actions.
  • Execute/write-back stage applies computation or memory transfer and commits updated architectural state.
  • Separation into stages makes control design cleaner and enables future pipeline optimization.

Typical register transfer micro-operations

Fetch phase: MAR <- PC MDR <- M[MAR] IR <- MDR PC <- PC + 1 Decode phase: Decode opcode and addressing mode in IR Execute phase (depends on opcode): AC <- AC + M[EA] (for ADD type) AC <- M[EA] (for LOAD type) M[EA] <- AC (for STORE type)

Topic 2: Register Roles (PC, IR, MAR, MDR, AC, GPR)

CPU registers are very small and very fast storage locations inside CPU used for instruction execution control and data manipulation.

Registers are the immediate working memory of CPU. Main memory is large but slower, so instruction execution is organized around fast register-level state transitions.

Control registers (PC, IR, MAR, MDR) coordinate instruction flow and memory transactions, while data registers (AC, GPRs) carry operands and results. Understanding this division is essential for solving register-trace PYQs accurately.

How registers cooperate during one instruction

  1. PC supplies next instruction address.
  2. MAR carries that address to memory interface.
  3. MDR receives fetched instruction word from memory.
  4. IR holds current instruction for decode/control logic.
  5. AC or GPRs provide operands and receive result depending on CPU organization.
Register Full form Main role Example action
PC Program Counter Holds address of next instruction PC <- PC + 1 after fetch
IR Instruction Register Holds currently fetched instruction IR <- MDR
MAR Memory Address Register Holds memory address for read/write MAR <- PC or MAR <- EA
MDR Memory Data Register Holds data moving to/from memory MDR <- M[MAR]
AC Accumulator Main arithmetic data register in accumulator machines AC <- AC + M[EA]
R0...Rn General Purpose Registers Hold operands/results in register-based CPUs ADD R1, R2, R3

Register-level fetch example

Initial: PC = 301 MAR <- PC => MAR = 301 MDR <- M[301] => MDR = 5941 IR <- MDR => IR = 5941 PC <- PC + 1 => PC = 302

Topic 3: Program Execution Trace Using Memory/Register Hex Values

Execution trace means tracking register and memory state step-by-step for each instruction cycle.

Trace problems test whether you understand dynamic execution, not only definitions. You must show how register values evolve from one cycle to the next under fetch and execute effects.

The best trace answers are deterministic and table-driven: start from initial state, apply fetch updates first, then execute updates, and carry forward the new state to the next row.

Trace-solving strategy

  • Keep one row per cycle and separate fetch result from execute result.
  • Update PC in every fetch unless control-flow instruction changes it.
  • Maintain hex formatting throughout if input is in hex.
  • When opcode semantics are assumed from course pattern, state that assumption once to avoid ambiguity.

Example A (PYQ 2022 style): three instruction cycles to add M[940] and M[941]

Given memory and instruction stream:

Address Content Meaning used in solution

302 2941 STORE 941
940 0004 Operand 1
941 0003 Operand 2
Exact opcode mapping may vary by textbook machine model. In this exam pattern, this sequence is treated as LOAD -> ADD -> STORE to realize M[941] = M[940] + M[941].
Cycle Fetch result Execute result State after cycle
1 IR=1940, PC=301 LOAD 940 -> AC=0004 AC=0004, IR=1940, PC=301

3 IR=2941, PC=303 STORE 941 -> M[941]=0007 AC=0007, IR=2941, PC=303, M[941]=0007

Example B (PYQ 2019 style): if PC = 301, find AC and IR

With PC=301, fetch reads instruction at address 301. So IR becomes 5941. AC at this point comes from previously executed first instruction (LOAD first operand), so AC=0004 in the standard sequence interpretation.

Topic 4: Three CPU Organizations

The three common CPU organizations are Accumulator, General Register, and Stack organization.

CPU organization decides operand placement model and instruction style. So instruction format, decoding complexity, and execution flexibility are directly influenced by organization choice.

Accumulator designs are compact but less flexible, general-register designs support high performance through explicit operand control, and stack designs reduce explicit addressing but rely on stack discipline.

Design trade-off intuition

  • Accumulator: smaller instruction width, but frequent memory traffic because AC is implicit center.
  • General register: more explicit fields, better compiler optimization opportunities.
  • Stack: compact zero-address operations, useful in expression evaluation models.

Accumulator organization

Single main register AC participates in almost all arithmetic operations.

Example: ADD A => AC <- AC + M[A]

General register organization

Multiple registers hold operands and results; flexible and fast.

Example: ADD R1, R2, R3

Stack organization

Operands are implicitly taken from stack top; zero-address operations common.

Example: PUSH A, PUSH B, ADD

Organization vs instruction style

Instruction style Likely organization Reason
ADD R1, A, B General register (3-address) Explicit destination and two sources
LOAD A Accumulator (1-address) Implicit AC as destination
PUSH AX Stack Stack pointer based push operation
ADD R1, B General register (2-address) One explicit register destination/source
Exam Focus:
  • If instruction has many explicit registers, think general-register organization.
  • One-address arithmetic usually indicates accumulator machine.
  • Zero-address arithmetic usually indicates stack machine.

Topic 5: Addressing Modes and Effective Address (EA) Calculation

Addressing mode defines how CPU obtains operand address or immediate data from an instruction.

Addressing modes are the language that connects instructions with data location patterns. Different programs need constants, direct variables, pointers, arrays, and register operands, and one fixed addressing rule cannot handle all efficiently.

Effective address (EA) is the computed memory location actually used for operand access in memory-based modes. Clear distinction between instruction field and final EA is the key to scoring full marks in numericals.

Why multiple modes are necessary

  • Immediate mode supports constants without memory access.
  • Register modes support fastest operand access for arithmetic loops.
  • Indexed and register-indirect modes support arrays and pointers naturally.
  • Indirect mode provides flexibility for dynamic address references.

Core formulas

Direct: EA = A, Operand = M[EA] Indirect: EA = M[A], Operand = M[EA] Register indirect: EA = (R), Operand = M[EA] Indexed: EA = A + XR, Operand = M[EA] Immediate: Operand = A (no memory read for operand) Register: Operand = (R)

Addressing mode summary table

Mode How operand is obtained Speed Typical use
Direct Use address field directly Medium Simple memory reference
Indirect Address field points to another address Slower Pointer-like memory access
Register indirect Register contains memory address Fast Array/pointer traversal
Immediate Operand value is inside instruction Fastest Constants
Indexed Base address + index register Fast Array indexing
Register Operand from register Fastest Arithmetic pipelines

Topic 6: RISC vs CISC and RISC Characteristics

RISC uses smaller, simpler instruction set and emphasizes pipeline-friendly design. CISC uses larger, more complex instruction set with richer instruction semantics.

RISC and CISC represent different design philosophies. RISC shifts complexity toward compiler and regular hardware pipeline, while CISC attempts to pack richer behavior into individual instructions to reduce instruction count and improve code density.

In practical systems, modern processors often mix ideas: many CISC processors internally break complex instructions into simpler micro-operations, and many RISC designs add extensions for workload efficiency. So comparison should be written as trade-off, not absolute winner/loser.

How to write a strong comparison answer

  1. Define RISC and CISC in one line each.
  2. Use a feature table (format, memory model, pipeline, control).
  3. Add one balanced conclusion: suitability depends on design goals such as performance, power, and code density.
Feature RISC CISC
Instruction complexity Simple Complex
Instruction length Usually fixed Often variable
Execution model Mostly single-cycle style for many ops Multi-cycle complex ops common
Memory access style Load/store architecture Memory operands in many instructions
Control implementation Often hardwired Historically microprogrammed emphasis
Pipeline friendliness High Lower than RISC generally
Code density Can be lower Often higher

RISC characteristics (exam points)

  • Simple instruction set and simple addressing modes.
  • Large register file.
  • Load/store architecture for memory operations.
  • Fixed instruction format supports efficient decode.
  • Designed for high pipeline efficiency.
Exam Focus:
  • For comparison question, present differences in table form for better marks.
  • Always mention load/store idea while writing RISC characteristics.

Topic 7: Basic MIPS Instruction Translation and Assembly Conversion

MIPS ISA basics: R-type for register arithmetic, I-type for immediate/memory/branch, J-type for jump.

MIPS translation questions connect ISA theory to binary encoding. You must map symbolic assembly fields to bit positions precisely, then present binary and hex forms.

Accuracy depends on systematic workflow: identify instruction type, fill fields in correct order, convert decimal immediates/register numbers to fixed-width binary, then combine and convert to hex.

Encoding workflow for machine translation

  1. Identify format (R, I, or J).
  2. Write field sequence from format template.
  3. Substitute opcode, registers, immediate/funct values.
  4. Ensure field bit-widths are correct.
  5. Join into 32-bit binary and convert to hexadecimal.

Assembly-conversion reasoning

For high-level constructs (loops/conditions), first model control flow using labels and branch condition direction. Then map array indexing into byte address arithmetic (index x word-size + base), and finally place load/compare/update instructions in correct order.

PYQ 2019 Q3(b): Given memory table and PC=301, find AC and IR.

Exam-ready answer:

Step 1: IR is loaded from memory location pointed by PC. So with PC=301, IR=M[301]=5941.
Step 2: AC reflects result of previously executed instruction. In this standard sequence (LOAD first operand then ADD), first instruction loads 0004 into AC.

Hence: IR=5941 and AC=0004 (before executing IR=5941).

If question context explicitly provides different opcode semantics, apply that map first. In this course pattern, the above interpretation is accepted.

2 (5941) PC=302, IR=5941 AC=0007
3 (2941) PC=303, IR=2941 M[941]=0007, AC=0007

PYQ 2020 A(b) and 2023 Q8(c): Need of addressing modes and explain any two.

Model answer:

Addressing modes are needed to provide flexibility in operand access, support arrays/pointers, reduce instruction count, and improve code efficiency.

Direct mode: instruction gives memory address directly (EA=A). Simple but less flexible.

Indexed mode: EA=A+XR. Very useful for arrays and loops because index changes while base remains fixed.

Other acceptable modes can also be explained: immediate, indirect, register, register-indirect.

PYQ 2020 A(d,e): Registers generally in processor and functions of MDR, IR, PC, MAR.

Model answer:

  • PC: stores next instruction address.
  • IR: stores current instruction under execution.
  • MAR: stores memory address for current memory operation.
  • MDR: stores data read from memory or to be written to memory.

Other common registers: AC, general-purpose registers, stack pointer, status/flag register.

PYQ 2020 B(b) and 2022 Q3(b): Translate lw/add/sub instructions into machine instructions.

PYQ 2020 while-loop conversion: while(A[i] == m) i = i+1;

Model MIPS solution:

# Assume: $s0 = base(A), $s1 = i, $s2 = m Loop: sll $t0, $s1, 2 add $t0, $t0, $s0 lw $t1, 0($t0) bne $t1, $s2, Exit addi $s1, $s1, 1 j Loop Exit:

Explanation: shift-left by 2 multiplies index by 4 (word size), then add base address, load A[i], compare with m, and increment i only when condition holds.

PYQ 2022 Q8(a): Distinguish between CISC and RISC computer.

Model answer: Provide a compact table with at least 5 points:

  • Instruction complexity: simple in RISC, complex in CISC.
  • Instruction length: fixed in RISC, variable common in CISC.
  • Addressing modes: fewer in RISC, many in CISC.
  • Pipeline support: easier in RISC.
  • Memory model: load/store in RISC; memory operands in many CISC instructions.

Final line: modern systems may combine ideas; difference is design philosophy, not strict superiority.
